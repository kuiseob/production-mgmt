## 1. 시스템 개요

### 1.1 프로그램 목적
**서진정밀**은 기계부품을 **수주생산(MTO, Make-to-Order)** 방식으로 제조하는 정밀가공 전문 업체입니다.
이 시스템은 다음 업무를 통합 관리합니다:

- 고객 **수주 등록 및 관리**
- 공정별 **작업지시 자동 생성** (일반CNC + 복합CNC)
- 일자별 **생산실적 수집**
- **품질검사** 결과 등록
- **출하 등록** 및 거래명세서 발행
- 일/월/연간 **통계 분석**
- **보고서** 인쇄 (HP, Samsung 등 모든 프린터)

### 1.2 핵심 특징
| 항목 | 내용 |
|---|---|
| **업무 방식** | MTO (Make-to-Order, 주문생산) |
| **공정 종류** | 일반CNC, 복합CNC (2종) |
| **보유 설비** | 일반CNC 13대 + 복합CNC 15대 = **총 28대** |
| **데이터 저장** | SQLite (production.db, 단일 파일) |
| **GUI** | Python tkinter (Mac/Windows 호환) |
| **그래프** | matplotlib (한글 자동 폰트) |
| **프린터** | HP, Samsung 등 시스템 설치된 모든 프린터 |
| **로그인** | 없음 (자동 admin 권한 진입) |

### 1.3 시스템 구성도

```
┌──────────────────────────────────────────┐
│         SEO JIN PRECISION System         │
├──────────────────────────────────────────┤
│                                          │
│  [GUI Layer]      tkinter (Python)       │
│       ↕                                  │
│  [Logic Layer]    Python 비즈니스 로직   │
│       ↕                                  │
│  [Data Layer]     SQLite production.db   │
│       ↕                                  │
│  [Output Layer]   HTML/CSV/PNG/Print     │
│                                          │
└──────────────────────────────────────────┘
```

## 2. 설치 및 실행

### 2.1 시스템 요구사항
- **OS**: Windows 10/11, macOS 12+, Linux (Ubuntu 22+)
- **메모리**: 최소 4GB, 권장 8GB
- **저장공간**: 100MB
- **화면 해상도**: 최소 1100×680, 권장 1340×820 이상
- **인터넷**: 다운로드 시에만 필요

### 2.2 첫 실행

**Windows**
```cmd
1. https://github.com/kuiseob/production-mgmt/releases 에서

2. 파일 우클릭 → 속성 → 차단 해제 체크 → 확인
3. 더블클릭 실행
```

**macOS**
```bash
# 한 번만: matplotlib 설치 (그래프 사용 시)
/opt/homebrew/bin/python3.13 -m venv ~/.prod_venv
~/.prod_venv/bin/pip install matplotlib

# 실행
~/.prod_venv/bin/python /Users/kuiseob/production-mgmt/production_app.py
```

### 6.2 📋 수주 관리

신규 주문을 등록하고 수정/삭제하는 핵심 화면입니다.

#### 입력 항목

| 항목 | 필수 | 형식 | 예시 |
|---|---|---|---|
| 주문번호 | ✓ | 자동 생성 (수정 가능) | `SJ-20260428-001` |
| 고객사 | ✓ | 드롭다운 선택 | 기아자동차 |
| 생산품명 | ✓ | 드롭다운 선택 | sdc80 case |
| 수량 | ✓ | 정수 | 500 |
| 수주일 | ✓ | YYYY-MM-DD | 2026-04-28 |
| 납기일 | ✓ | YYYY-MM-DD | 2026-05-30 |
| 비고 | - | 자유 텍스트 | 긴급, 표면처리 |

#### 3개 액션 버튼

| 버튼 | 색상 | 동작 |
|---|---|---|
| 🟢 **수주 등록** | 녹색 (#2E7D32) | 신규 주문 INSERT (중복 주문번호 자동 검사) |
| 🟠 **수주 수정** | 주황 (#E65100) | 선택한 주문 정보 변경 |
| 🔴 **수주 삭제** | 빨강 (#C62828) | 주문 + 연관 작업/검사/출하 함께 삭제 |

#### 사용 방법

**A. 신규 등록**
1. 폼에 모든 필수 항목 입력
2. **수주 등록** 클릭
3. → "수주 [SJ-...] 등록 완료!" 알림

**B. 수정**
1. 하단 **목록에서 행 클릭** → 폼이 자동으로 데이터 채워짐
2. 변경할 값 수정
3. **수주 수정** 클릭 → 확인 → 변경됨

**C. 삭제**
1. 목록에서 행 클릭
2. **수주 삭제** 클릭
3. 연관 데이터 건수 확인 (예: "작업지시 2건 / 출하 1건 함께 삭제")
4. **예** 클릭 → 모두 삭제

#### 주의사항
- 주문번호 중복 시 거부됩니다 (다른 번호 사용)
- 삭제는 되돌릴 수 없습니다 — 신중히 진행
- 출하완료된 수주도 삭제 가능 (관련 출하 데이터도 같이 삭제됨)

### 6.3 📅 생산계획

수주를 작업지시(WO)로 변환합니다.

#### 동작 원리
한 수주를 선택하면 **공정 수만큼 WO가 자동 생성**됩니다:
- 일반CNC WO 1건 (`WO-...-01`)
- 복합CNC WO 1건 (`WO-...-02`)

#### 사용 방법
1. 목록에서 수주 한 건 **선택** (한 번 클릭)
2. **작업지시 자동 생성** 버튼 클릭 (파란색)
3. → "작업지시 2건 생성 완료! (일반CNC, 복합CNC)" 알림
4. 수주 상태: 접수 → **진행중**
5. WO수 컬럼에 "2" 표시됨

#### 추가 생성
이미 WO가 있는 수주를 다시 선택하고 자동 생성을 클릭하면:
- "이미 N건의 작업지시가 있습니다. 추가로 생성할까요?" 확인
- **예** 시 추가 2건 더 생성 (총 4건)

#### 표시 컬럼
| 컬럼 | 내용 |
|---|---|
| 주문번호 | SJ-YYYYMMDD-NNN |
| 고객사 | 거래처명 |
| 생산품명 | 부품명 |
| 수량 | 주문 수량 |
| 납기일 | YYYY-MM-DD |
| 상태 | 접수 / 진행중 |
| WO수 | 생성된 WO 개수 |

### 6.4 🔧 작업지시

설비/담당자를 배정하고 작업 진행을 관리하며, 잘못된 WO를 수정/삭제합니다.

#### 입력 항목

| 항목 | 설명 |
|---|---|
| 설비 배정 | **모든 28개 설비** 표시 (공정과 무관, 자유 선택)<br>형식: `CNC-08 - LYNX2100M-1 (일반CNC)` |
| 담당자 | 작업자 선택 |
| 계획수량 | WO의 계획 수량 수정 (행 클릭 시 자동 채워짐) |

#### 5개 액션 버튼 (한 줄 우측 정렬)

| 🟠 **WO 수정** | 주황 | 계획수량 변경 |
| 🔴 **WO 삭제** | 빨강 | WO + 생산실적 함께 삭제 |

#### 표준 작업 흐름
1. 목록에서 작업 행 **선택** (한 번 클릭)
2. 상단 **선택** 라벨에 정보 표시 (예: "WO-... (sdc80 case / 일반CNC / 계획 500)")

4. 실제 작업 시작 시 → **작업 시작**

## 10. 데이터 저장 및 백업

### 10.1 데이터베이스 위치

- **위치**: 프로그램 실행 폴더

### 10.2 모든 데이터가 한 파일에
- 수주, 작업지시, 생산실적, 검사, 출하 등 **모든 데이터**가 `production.db` 한 파일에 저장됩니다.
- 백업 = 이 파일 하나만 복사하면 끝

### 10.3 수동 백업
**Windows**
```cmd
copy production.db production_2026-04-28.db
```

**macOS**
```bash
cp production.db production_2026-04-28.db
```

### 10.4 자동 백업 (스크립트)
```bash
#!/bin/bash
# backup.sh
cd /Users/kuiseob/production-mgmt
DATE=$(date +%Y%m%d_%H%M)
cp production.db backups/production_$DATE.db
echo "백업 완료: production_$DATE.db"
```

### 10.5 데이터 이전
- DB 파일을 다른 PC로 복사 → EXE 또는 Python으로 실행 → 자동 인식
- 모든 데이터(수주/생산/검사/출하)가 그대로 이전됨

### 10.6 외장 백업 권장

## 14. 자주 묻는 질문 (FAQ)

### Q1. 로그인이 필요한가요?
A. 아니요. 별도의 로그인 절차 없이 자동으로 관리자 권한으로 진입합니다.

### Q2. 한글 입력 후 저장 버튼을 눌러도 "필수 입력" 오류가 나요.
A. 한글 IME 미커밋 상태일 수 있습니다. **Tab 키나 Enter 키**로 입력을 확정한 후 버튼을 눌러주세요. 최신 버전은 자동 커밋되도록 처리되어 있습니다.

### Q3. 수주를 잘못 등록했어요. 어떻게 수정/삭제하나요?
A.
- **수정**: 목록 행 클릭 → 폼 자동 채워짐 → 값 변경 → **수주 수정** 버튼
- **삭제**: 목록 행 클릭 → **수주 삭제** 버튼 → 확인

### Q4. 같은 코드의 고객사를 다시 등록할 수 없어요.
A. 한 번 삭제(소프트 삭제)된 코드를 다시 등록하면 **자동으로 복원/갱신 옵션**이 제공됩니다.

### Q5. 작업지시는 어떻게 만드나요?
A.
1. **수주 관리**에서 주문 등록
2. **생산계획**에서 해당 주문 선택 → **작업지시 자동 생성** 클릭
3. → 일반CNC + 복합CNC 2개 WO가 자동 생성됨

### Q6. 잘못 생성된 작업지시를 수정/삭제할 수 있나요?
A. 네. **🔧 작업지시** 메뉴에서:
- 행 클릭 → 계획수량 수정 → **WO 수정**
- 행 클릭 → **WO 삭제** (연관 생산실적 함께 삭제)

### Q7. 그래프를 PNG로 저장하고 싶어요.
A. **통계 (일/월/년)** 메뉴 → 탭 선택 → **그래프 저장** 버튼 → 파일 위치 지정

### Q8. 데이터를 Excel에서 열고 싶어요.
A. **통계** 메뉴에서 **CSV 저장** 버튼 → 생성된 CSV 파일을 Excel에서 더블클릭 (UTF-8 BOM 포함)

### Q9. 사이드바 메뉴가 잘 안 보여요.
A. 화면 해상도 1340×820 이상을 권장합니다. 작은 화면은 메뉴 폭(285px)을 차지하여 페이지 영역이 좁아질 수 있습니다.

### Q10. 프린터에서 인쇄가 안 돼요.
A.
1. Windows에서 프린터가 정상 설치되어 있는지 확인
2. 인쇄 다이얼로그에서 올바른 프린터 선택
3. A4 용지 사용 권장
4. 프린터 드라이버 최신 버전 사용

### Q11. DB 파일을 백업하고 싶어요.
A. 프로그램 종료 후 `production.db` 파일을 다른 위치로 복사하면 됩니다. (10장 참조)

### Q12. 거래명세서가 비어있어요.
A. 먼저 **출하 등록**을 해야 합니다. 검사 합격 → 출하 등록 → 거래명세서가 생성됩니다.

### Q13. 그래프 분석에서 차트가 1개만 보여요.
A. 상단 **유형 ▼ 드롭다운**에서 6종 중 하나 선택. 각 선택마다 자동으로 차트가 전환됩니다.

### Q14. macOS에서 그래프가 □□로 깨져 보여요.
A. 한글 폰트가 자동으로 감지됩니다 (AppleGothic). 만약 깨진다면 macOS의 **시스템 폰트** 설정 확인.

### Q15. 통계의 연간 탭에서 오류가 나요.
A. 최신 버전(2026-04-28 이후)에서 수정되었습니다. GitHub Releases에서 최신 EXE 다운로드.

## 15. 문제 해결 (Troubleshooting)

### 15.1 프로그램이 실행되지 않을 때

**Windows**
- EXE 우클릭 → 속성 → 차단 해제
- SmartScreen "추가 정보" → 실행
- 백신 프로그램의 격리 폴더 확인

**macOS**
```bash
# Python 확인
/opt/homebrew/bin/python3.13 --version

# 직접 실행
/opt/homebrew/bin/python3.13 ~/production-mgmt/production_app.py
```

### 15.2 데이터베이스 오류 (DB locked)
- 동일한 DB 파일을 여러 프로그램에서 열고 있을 수 있습니다.
- 모든 프로그램 종료 → 재실행

### 15.3 화면이 깨져 보일 때
- 윈도우 크기 조정 (최소: 1100×680)
- 프로그램 종료 → 재실행
- 다른 모니터에서 실행 시 해상도 차이 확인

### 15.4 그래프가 표시되지 않을 때 (Mac)
matplotlib 설치 필요:
```bash
~/.prod_venv/bin/pip install matplotlib
```

### 15.5 프린터 선택 다이얼로그가 안 뜸 (Mac)
- macOS는 직접 프린터 선택 기능 없음 (Windows 전용)
- 브라우저 인쇄 다이얼로그(Ctrl+P/Cmd+P)에서 선택 가능

### 15.6 한글이 깨질 때
- **GUI**: macOS는 자동으로 시스템 한글 폰트 사용
- **그래프**: AppleGothic / Apple SD Gothic Neo 자동 감지
- **인쇄**: 브라우저가 시스템 한글 폰트 사용

### 15.7 입력값이 저장되지 않을 때
- IME(한글 입력기) 상태 확인
- **Tab 키** 또는 **Enter 키**로 입력 확정 후 저장
- 입력 폼 다시 클릭하여 포커스 확인

### 15.8 EXE 파일이 손상되었다고 나올 때
